# 全栈 AI 前端 30 天刷题路线

## 使用说明
- 每天学习时长建议 `2.5-3.5` 小时：`60% 面试题理解 + 25% 手写/设计 + 15% 复盘输出`。
- 每天至少产出：`1 份书面答案`、`1 段手写代码`、`1 个项目迁移思路`。
- 按照目录中的 `DayXX/README.md` 执行，完成后在“完成状态”打勾。
- 配套文件：
  - `评分表.md`：每天量化打分与短板追踪
  - `参考答案框架.md`：每天题目的结构化答题骨架

## 面试目标画像（5-8 年）
- 能从“会用”升级到“讲清原理 + 说明边界 + 给出工程权衡”。
- 能把前端问题延展到后端、数据库、缓存、AI 业务链路。
- 能独立完成系统设计题：高可用、可观测、可扩展、可回滚。

## 每日统一答题模板
1. **基础原理**：定义、机制、关键流程图（可文字化）。
2. **深度解析**：设计权衡、常见坑、性能/安全影响。
3. **拓展理解**：和上下游系统关系、替代方案、演进路径。
4. **实战输出**：手写题或场景题，包含边界与异常处理。

## 30 天主题总览

### 第 1 周：前端底层与浏览器核心
- Day01: JS 执行机制、事件循环、微任务/宏任务
- Day02: 作用域链、闭包、this/new/bind 实现原理
- Day03: 原型链、继承模型、对象描述符与 Proxy
- Day04: Promise/A+、async/await、错误传播与取消
- Day05: 浏览器渲染流水线、重排重绘、性能指标
- Day06: HTTP/1.1/2/3、缓存协商、TCP/TLS 基础
- Day07: 安全体系（XSS/CSRF/CORS/CSP）与防护

### 第 2 周：React/工程化/质量体系
- Day08: React Fiber、调和、优先级与批处理
- Day09: Hooks 原理、闭包陷阱、状态建模
- Day10: 状态管理（Redux/Zustand）与一致性设计
- Day11: 前端工程化（Vite/Webpack、Tree Shaking、分包）
- Day12: TypeScript 类型系统（泛型、条件类型、类型体操）
- Day13: 测试体系（单测/集成/E2E）与可测试架构
- Day14: 可观测性（日志、埋点、前端监控）与故障定位

### 第 3 周：全栈能力与后端基础
- Day15: Node.js 事件驱动、流、背压、性能诊断
- Day16: 服务端 API 设计（REST/GraphQL）与契约治理
- Day17: 数据库（MySQL 索引/事务）与 SQL 优化
- Day18: Redis 缓存模型、穿透击穿雪崩、分布式锁
- Day19: 鉴权体系（JWT/Session/OAuth2）与安全边界
- Day20: 微服务/网关/BFF 与前后端职责划分
- Day21: 消息队列、异步任务、幂等与最终一致性

### 第 4 周：AI 全栈场景与高阶面试实战
- Day22: LLM 基础、Prompt 设计、上下文窗口与成本
- Day23: RAG 全链路（切片、召回、重排、评测）
- Day24: AI 应用架构（多 Agent、工具调用、权限隔离）
- Day25: AI 产品性能（延迟、并发、流式传输、降级）
- Day26: AI 安全（提示注入、数据泄露、输出风控）
- Day27: 系统设计题专项（高并发 AI 问答平台）
- Day28: 项目深挖题专项（STAR + 指标量化 + 复盘）
- Day29: 手写题冲刺（日常高频 + 全栈综合）
- Day30: 全真模拟面试（技术 + 项目 + 场景 + 复盘）

## Day01~Day30 题型要求（每天都要覆盖）
- **基础原理题（2 题）**：解释机制 + 关键 API/流程。
- **深度解析题（2 题）**：谈边界、性能、错误处理、工程实践。
- **拓展理解题（1 题）**：跨层延展到后端/AI/架构。
- **手写或场景题（1 题）**：限时 30-45 分钟。

## 交付标准（自测）
- 能在 `3-5` 分钟讲清核心概念，不依赖背稿。
- 能举出至少 `2` 个线上问题案例和排查链路。
- 每道题都能回答：为什么这样设计？还有什么方案？代价是什么？
